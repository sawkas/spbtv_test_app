Напишите на Ruby сервис, реализующий REST API со следующими функциями:

1. Прием от клиентов уведомления о том, что клиент смотрит видео.
  Параметры:
  * customer_id - идентификатор пользователя;
  * video_id - идентификатор видео, которое смотрит пользователь;

2. Отвечает на запрос, сколько видео потоков смотрит пользователь в данный момент.
  Параметры:
  * customer_id - идентификатор пользователя;

3. Отвечает на вопрос, сколько пользователей смотрят данное видео в текущий момент.
  Параметры:
  * video_id - идентификатор видео;

Дополнительные сведения:
- клиенты посылают уведомления о просмотре видео каждые 5 секунд, пока работает плеер. Если в течение 6 или более секунд  уведомление от пользователя не пришло, то считается, что просмотр этого видео завершен;
- клиентов много, и их не интересует ответ на уведомление о просмотре;
- запросы 2 и 3 проходят существенно реже, нежели запрос 1;
- ответ на запрос 2 должен быть максимально актуальным;
- ответ на запрос 3 может отставать и отражать ситуацию не на текущий момент, а, допустим, на минуту назад;

Требования:
- можно использовать любой фреймворк, либо ничего не использовать;
- все данные должны храниться в памяти Ruby приложения, использовать внешнее хранилище не нужно;
- однако необходимо предусмотреть возможность подключения внешнего хранилища через адаптер (например, хранилищем может быть Redis).
- код проекта должен быть размещен на Github;

По желанию:
- сервис отвечает на вопрос, сколько запросов было обработано за последнюю минуту;
- оформить решение в виде Ruby Gem
- провести нагрузочное тестирование в любом доступном окружении;
- дополнительно реализовать адаптер, использующий Redis вместо памяти приложения
